# Skip Go 문서

Skip Go 프로젝트의 종합 문서 모음입니다. 이 문서들은 **15년차 시니어 개발자가 후배를 위해 작성한 실무 중심의 가이드**입니다.

## 문서 목록

### 1. [프로젝트 개요](./PROJECT_OVERVIEW.md)

**프로젝트의 목적과 배경을 이해하기 위한 문서**

Skip Go가 무엇인지, 왜 만들어졌는지, 어떤 문제를 해결하는지를 설명합니다.

**대상**: 프로젝트를 처음 접하는 모든 사람

**주요 내용**:

- 프로젝트의 목적과 배경
- Skip Go가 해결하는 문제
- 핵심 기능 및 가치
- 기술적 목표
- 주요 사용 사례

**읽는 시간**: 15-20분

---

### 2. [아키텍처 문서](./ARCHITECTURE.md)

**전반적 및 세부 아키텍처를 체계적으로 설명하는 문서**

프로젝트의 전체 구조, 데이터 플로우, 상태 관리, 컴포넌트 구조 등을 상세히 설명합니다.

**대상**: 프로젝트 구조를 깊이 이해하고 싶은 개발자

**주요 내용**:

- 아키텍처 개요 및 설계 원칙
- 패키지별 상세 아키텍처
- 데이터 플로우 상세 분석 (초기화, 스왑, 트랜잭션 실행)
- 상태 관리 아키텍처 (Jotai 원자 구조)
- 컴포넌트 아키텍처
- API 통합 아키텍처
- 지갑 통합 아키텍처
- 트랜잭션 실행 아키텍처
- 에러 처리 아키텍처

**읽는 시간**: 30-45분

- UI 컴포넌트 작업 (실무)

- 테스트 작성 (실무)

## 빠른 참조

### 주요 파일 위치

- **위젯 진입점**: `packages/widget/src/widget/Widget.tsx`
- **클라이언트 진입점**: `packages/client/src/index.ts`
- **라우팅**: `packages/widget/src/widget/Router.tsx`
- **상태 관리**: `packages/widget/src/state/`
- **API 통합**: `packages/client/src/api/`
- **트랜잭션 실행**: `packages/client/src/public-functions/executeRoute.ts`

### 주요 개념

- **모노레포**: Yarn Workspaces를 사용한 다중 패키지 관리
- **상태 관리**: Jotai 원자 기반 상태 관리
- **지갑 통합**: Provider 패턴을 통한 멀티체인 지갑 지원
- **API 통합**: Swagger 기반 자동 생성 클라이언트
- **Shadow DOM**: 스타일 격리를 위한 Shadow DOM 사용

### 주요 원자 (Atoms)

- `sourceAssetAtom`: 소스 자산 정보
- `destinationAssetAtom`: 대상 자산 정보
- `skipRouteAtom`: 현재 라우트 정보 (비동기)
- `walletsAtom`: 연결된 지갑 정보
- `currentPageAtom`: 현재 페이지

### 주요 Hooks

- `useCreateCosmosWallets`: Cosmos 지갑 생성
- `useCreateEvmWallets`: EVM 지갑 생성
- `useCreateSolanaWallets`: Solana 지갑 생성
- `useGetBalance`: 잔액 조회
- `useFetchAllBalances`: 모든 체인 잔액 조회

## 문서 특징

### 실무 중심

- **인수인계 관점**: 인수인계 받은 개발자가 바로 작업할 수 있도록 작성
- **단계별 가이드**: 각 작업을 단계별로 상세히 설명
- **코드 예시**: 실제 사용 가능한 코드 예시 포함
- **문제 해결**: 일반적인 문제와 해결 방법 포함

### 체계적 구조

- **목적별 문서 분리**: 각 문서가 명확한 목적을 가짐
- **단계별 학습**: 단계별로 학습할 수 있도록 구성
- **상호 참조**: 문서 간 상호 참조로 연결

### 상세한 설명

- **15년차 시니어 관점**: 후배를 위한 상세한 설명
- **왜 그렇게 하는지 설명**: 단순히 방법만이 아닌 이유도 설명
- **실무 팁**: 실제 개발에서 유용한 팁 포함

## 문서 업데이트

문서는 프로젝트와 함께 발전합니다.

- **개선 사항**: 문서 개선 제안은 PR로 제출
- **오류 수정**: 오류 발견 시 즉시 수정
- **최신화**: 프로젝트 변경 시 문서도 함께 업데이트
